# 🧭 LECTURE-2 OVERVIEW — দুটো নতুন Question Type (পদ্ধতি + কৌশল)
### English content · বাংলায় ব্যাখ্যা · for Md. Ariful Islam
> Lecture-1-এ ছিল MCQ, Short Answer, Summary, Sentence Completion (উত্তর = একটা শব্দ/একটা option খোঁজা)।
> Lecture-2-এ আছে **দুটো "বড় ছবি" question type** — এখানে উত্তর = পুরো paragraph-এর **মূল ভাব** বা পুরো statement-এর **সত্য-মিথ্যা** বোঝা। এগুলোতে non-native-রা সবচেয়ে বেশি মার্ক হারায়, তাই মন দিয়ে পড়ো।

---

## 🗺️ এই দুটো type-এর মূল পার্থক্য (আগে এটা গেঁথে নাও)

| | **Q-Type 5 — Matching** | **Q-Type 6 — T/F/NG · Y/N/NG** |
|---|---|---|
| কী চায় | paragraph ↔ heading / statement ↔ paragraph **মেলানো** | একটা statement passage-এর সাথে **মেলে / উল্টো / নেই** ঠিক করা |
| উত্তর | একটা number (i, ii…) বা letter (A, B…) | TRUE/FALSE/NOT GIVEN (বা YES/NO/NOT GIVEN) |
| পড়ার ধরন | **gist** (পুরো অনুচ্ছেদের মূল ভাব) | **detail** (নির্দিষ্ট লাইন মিলিয়ে দেখা) |
| সবচেয়ে বড় ফাঁদ | একই শব্দ দুই heading-এ থাকে; **মূল ভাব** ধরো, শব্দ নয় | **Not Given** কে False ভেবে ফেলা |

# ১️⃣ MATCHING HEADINGS — paragraph-এর "main idea" ধরার খেলা

## 🎯 Mentor's method (এই ৬ ধাপে কাজ করো)
1. **আগে heading-এর তালিকা পড়ো না বেশি** — প্রথমে **paragraph-টা পড়ো** (বিশেষ করে প্রথম ও শেষ বাক্য = topic sentence + conclusion)।
2. নিজেকে জিজ্ঞেস করো: **"এই অনুচ্ছেদটা মূলত কী নিয়ে?"** — এক বাক্যে নিজের ভাষায় বলো।
3. এবার তালিকায় গিয়ে তোমার এক-বাক্যের সাথে **সবচেয়ে মিলে যাওয়া** heading বাছো।
4. একটা heading একবারই ব্যবহার হয় (যদি না অন্যথা বলা থাকে) — ব্যবহৃত heading **কেটে দাও**।
5. **Example** দেওয়া থাকলে (যেমন Module-1-এ D=iv দেওয়া) — সেই heading তালিকা থেকে বাদ; ওটা আর অন্য কোথাও বসবে না।
6. শেষে কঠিন/বাকি paragraph-গুলোর জন্য **বাকি থাকা heading** দিয়ে elimination করো।

> 🔑 **সোনালি সূত্র:** Heading = পুরো অনুচ্ছেদের **ছাতা (umbrella)**, একটা detail নয়। একটা heading যদি অনুচ্ছেদের মাত্র একটা বাক্য ঢাকে কিন্তু পুরোটা নয় → ❌ ভুল।

## ⚠️ Matching Headings-এর ৪ ফাঁদ
- **Detail trap:** heading অনুচ্ছেদের একটা ছোট তথ্য তুলে ধরে, মূল ভাব নয়। (পুরোটা ঢাকছে কিনা দেখো।)
- **Same-word trap:** heading-এ অনুচ্ছেদের হুবহু একটা শব্দ আছে — কিন্তু ভাব মেলে না। শব্দ-ম্যাচ ≠ অর্থ-ম্যাচ।
- **Too-broad / too-narrow:** heading হয় অতি-সাধারণ (সব অনুচ্ছেদেই খাটে) বা অতি-সংকীর্ণ। মাপমতো একটাই বাছো।
- **First-sentence trap:** সবসময় topic sentence প্রথম বাক্যে থাকে না — কখনো শেষ বাক্যে বা "However…/Hence…"-এর পরে আসল ভাব।

# ২️⃣ MATCHING INFORMATION — statement ↔ paragraph খোঁজার খেলা

Heading matching-এর কাছাকাছি, কিন্তু এখানে heading নয় — তোমাকে একটা **specific তথ্য/যুক্তি** কোন paragraph-এ আছে সেটা খুঁজতে হবে।

## 🎯 Mentor's method (৫ ধাপ)
1. **statement-টা ভালো করে পড়ো**, এর "তথ্য-ধরন" বুঝো — এটা কি একটা *কারণ? উদাহরণ? তুলনা? মতামত? সীমাবদ্ধতা?* (Lecture-2-এর statement-গুলো ঠিক এভাবেই লেখা: "a compromise…", "an independent opinion…", "a comparison…")।
2. statement-কে **paraphrase** ধরে নাও — passage-এ হুবহু শব্দ থাকবে না, **ধারণাটা** খোঁজো।
3. প্রতিটা paragraph **scan** করো ওই ধারণার জন্য।
4. পেলে → letter বসাও। **NB "use any letter more than once"** থাকলে একই paragraph একাধিক প্রশ্নের উত্তর হতে পারে — ভয় পেও না।
5. ❗ Matching Information-এ প্রশ্ন **passage-এর ক্রমে আসে না** — তাই পুরো passage মাথায় রেখে খুঁজতে হয়।

> 🔑 **মূল পার্থক্য মনে রাখো:** Heading = অনুচ্ছেদের **পুরো ভাব**। Information = অনুচ্ছেদের ভেতরের **একটা নির্দিষ্ট তথ্য**। তাই Information matching-এ একটা ছোট বাক্যও পুরো উত্তর হতে পারে।

# ৩️⃣ TRUE/FALSE/NOT GIVEN ও YES/NO/NOT GIVEN — সবচেয়ে গুরুত্বপূর্ণ type

## পার্থক্য আগে বুঝো
| | **TRUE / FALSE / NOT GIVEN** | **YES / NO / NOT GIVEN** |
|---|---|---|
| কী যাচাই | passage-এর **তথ্য / facts** | লেখকের **মত / view / claim / opinion** |
| TRUE/YES | statement তথ্য/মতের সাথে **মেলে** | statement লেখকের মতের সাথে **মেলে** |
| FALSE/NO | statement **উল্টো** বলছে | statement লেখকের মতের **বিপরীত** |
| NOT GIVEN | passage-এ **নেই** | লেখকের মত **জানা যায় না** |

## 🎯 Mentor's method (৬ ধাপ) — মুখস্থ করো
1. statement-টা পড়ো, এর **keyword** ধরো (নাম, সংখ্যা, technical শব্দ — যেগুলো paraphrase হয় না)।
2. সেই keyword passage-এ **scan** করো — statement-গুলো সাধারণত **passage-এর ক্রমে** আসে (Q1 আগে, Q2 পরে)।
3. পেলে ওই ১-২ বাক্য **ধীরে পড়ো**।
4. **তিনটা প্রশ্ন নিজেকে করো:**
   - passage কি statement-টা **নিশ্চিত করছে**? → **TRUE / YES**
   - passage কি statement-এর **উল্টো** বলছে? → **FALSE / NO**
   - passage কি এ বিষয়ে **কিছুই বলছে না**? → **NOT GIVEN**
5. **Quantity/qualifier-এ চোখ রাখো:** *some / many / most / all / only / usually / sometimes* — statement আর passage-এর পরিমাণ এক কিনা মেলাও। ("many" বললে passage, "most" বললে statement → অনেক সময় Not Given/False)।
6. **Negative words-এ থামো:** *no, not, never, rarely, hardly* — একটা "not" মিস করলে TRUE↔FALSE উল্টে যায়।

## ⚠️ এই type-এর ৫ ফাঁদ (এখানে non-native-রা সবচেয়ে বেশি মার্ক হারায়)
1. **False vs Not Given গুলিয়ে ফেলা** — সবচেয়ে বড় ভুল।
   - **FALSE** = passage **সরাসরি উল্টো** বলছে (contradiction)।
   - **NOT GIVEN** = passage **কিছুই বলছে না** (no information)।
   - 🧠 টেস্ট: "passage কি statement-এর বিপরীতটা প্রমাণ করে?" — হ্যাঁ হলে False, না হলে (চুপ থাকলে) Not Given।
2. **নিজের জ্ঞান/common sense** দিয়ে উত্তর — "এটা তো সত্যি হওয়ারই কথা" ভেবে TRUE দিও না। passage-এ লেখা না থাকলে **Not Given**।
3. **Extreme qualifier** — statement-এ *all/always/only/never* থাকলে আর passage *some/often* বললে → সাধারণত **False/Not Given**।
4. **"Fact" vs "theory/opinion" trap** — passage বলছে "one theory is…" (অনুমান), statement বলছে "It is a fact that…" → **False** (theory কে fact বললে ভুল)। [Frog Q10 এই ফাঁদ]।
5. **Half-true statement** — অর্ধেক passage-এর সাথে মেলে, অর্ধেক যোগ করা/বদলানো → পুরোটা না মিললে **TRUE নয়**।

## 🥇 Mentor's golden order (Lecture-2-এর সব type-এর জন্য)
1. **Matching Headings** পেলে → আগে passage-টা skim করে প্রতিটা অনুচ্ছেদের পাশে ২-৩ শব্দে নিজের label লেখো ("কারণ", "উদাহরণ", "ইতিহাস")। তারপর heading মেলাও।
2. **Matching Information** পেলে → statement-এর ধরন (কারণ/মত/তুলনা) বুঝে পুরো passage scan করো।
3. **T/F/NG · Y/N/NG** পেলে → keyword scan → ১-২ লাইন ধীরে পড়ো → তিন-প্রশ্ন টেস্ট → qualifier/negative মেলাও।
4. কঠিন প্রশ্ন **flag → guess → পরে ফেরো**। NG/F-এ আটকে গেলে: "passage কি উল্টোটা বলছে?" — না হলে NG।

> পরের ফাইলে (`02`–`04`) প্রতিটা exercise **ধাপে ধাপে** সমাধান করা — কোথায় উত্তর (locate), কেন এটাই উত্তর, কেন বাকিগুলো ভুল (trap)। Hooks ও তোমার common mistake → `06`।
